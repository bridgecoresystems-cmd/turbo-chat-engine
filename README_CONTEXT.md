# Проект: Turbo Chat Engine (BridgeCore Systems)
**Автор:** Батыр Акмурадов
**Цель:** Создание экстремально быстрого чат-фреймворка (движка) на Rust с минимальной задержкой (low latency) и максимальной пропускной способностью (high throughput).

## 1. Философия проекта
- **Никакого JSON:** Взаимодействие только через бинарный протокол (Protobuf).
- **Никакого Garbage Collector:** Использование Rust для детерминированного управления памятью.
- **Специализация:** Движок предназначен *только* для обмена сообщениями. Весь стандартный веб-функционал (HTML, сессии, формы) исключен для оптимизации.
- **Цель в бенчмарках:** Попадание в топ TechEmpower Web Framework Benchmarks.

## 2. Технологический стек
- **Язык:** Rust (последняя стабильная версия).
- **Рантайм:** `tokio` (асинхронный ввод-вывод).
- **Сетевой уровень:** WebSockets (на старте) с последующим внедрением QUIC/HTTP3.
- **Системная оптимизация:** Использование `io_uring` на Linux через `tokio-uring`.
- **Сериализация:** `Protobuf` (библиотека `prost`).
- **Транспортный слой (Pub/Sub):** `Redis` для маршрутизации между узлами.
- **Хранилище (Persistence):** `PostgreSQL` с использованием Batching Worker (оптовая запись пачками).

## 3. Архитектурные слои
1. **Connection Layer:** Удержание сотен тысяч активных WebSocket-соединений. Ожидаемое потребление: 2-5 КБ на соединение.
2. **Routing Layer:** Распределенная доставка сообщений через Redis Pub/Sub и локальные асинхронные каналы (mpsc/broadcast).
3. **Persistence Layer:** Фоновый воркер, который аккумулирует сообщения в памяти и делает Bulk Insert в Postgres (например, каждые 2 секунды или по достижении 1000 записей).

## 4. Текущая инфраструктура (Docker Compose)
- **Postgres 16-alpine** (Порт 5432).
- **Redis 7-alpine** (Порт 6379).

## 5. Ближайшие задачи для реализации
1. Инициализация проекта через `Cargo`.
2. Описание структуры сообщения в `proto/message.proto`.
3. Реализация базового WebSocket-сервера на `tokio` и `fastwebsockets`.
4. Настройка логики Broadcast-рассылки сообщений всем подключенным клиентам.
